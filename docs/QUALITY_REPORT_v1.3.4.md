> 基于 110 题实测数据 | 2026-06-21 | DeepSeek V4 Flash

## 一、评估方法

对 110 道测试题目逐一提取 AI 最终回复，按 9 个维度自动化评分，满分 11 分。

### 评分维度

| 维度 | 权重 | 检测方式 |
|------|:---:|---------|
| 有具体数据（分数/位次） | 2 | 正则匹配 `XXX分` / `XXX名` |
| 标注数据来源 | 2 | URL 链接 / "来源"关键词 |
| 有免责提醒 | 1 | "官方" "仅供参考" "考试院" "提醒" |
| 有表格呈现 | 1 | Markdown 表格 |
| 有趋势分析 | 1 | 多年份/趋势关键词 |
| 区分投档线/专业线 | 1 | 明确区分两者差异 |
| 有可行方案建议 | 1 | "冲稳保"/方案/推荐 |
| 回答长度 > 500 字 | 1 | 内容丰富度 |
| 回答长度 > 1500 字 | 1 | 内容深度 |

### 等级标准

| 等级 | 分数 | 含义 |
|:---:|:---:|------|
| **S** | 8-11 | 优秀——数据详实、来源清晰、有方案 |
| **A** | 6-7 | 良好——要点覆盖、基本完整 |
| **B** | 4-5 | 及格——有核心数据但缺深度 |
| **C** | 2-3 | 不足——信息简略或缺失关键维度 |
| **D** | 0-1 | 不合格——基本无有效信息 |

## 五、典型样例

### S 级 — "华东五校2025年各省录取分对比"（11分，3,313字）

**覆盖维度**: 分数位次 ✅ 来源 ✅ 表格 ✅ 趋势 ✅ 投档/专业区分 ✅ 方案 ✅ 深长 ✅

典型输出特征：
- 5 校 × 多省对比表格
- 分省份逐年数据
- 2023-2025 趋势箭头
- 明确标注"以上数据来自高考100、6617.com，以各省考试院公布为准"
- 最后给出"报哪所取决于专业偏好"的策略建议

### A 级 — "计算机专业2026年还值得学吗"（6分，~900字）

**缺失维度**: 无表格、无来源标注

典型输出特征：
- 结论先行："仍然值得，但要选对赛道"
- AI vs 计算机对比清晰
- 引用知识库中人工智能专业画像（数学要求、课程难度）
- 提到薪资范围但未标注数据来源

### C 级 — "强基计划适合什么学生"（3分，~300字）

**缺失维度**: 无数据、无表格、无来源、无趋势

典型输出特征：
- 简要说明强基计划的定位
- "适合对基础学科有浓厚兴趣的学生"
- 未提供具体院校案例或录取数据
- 未建议用户进一步查询官方渠道

## 六、核心发现

### 6.1 强项

1. **趋势分析**（88%）—— 系统提示词"优先参考 2025 其次 2024"被严格遵守，几乎所有回复都做了年份对比
2. **方案导向**（81%）—— 张雪峰风格"先给结论再给选择"落实到位
3. **数据呈现**（73%）—— search_web 可靠地返回了真实录取分数和位次
4. **表格使用**（77%）—— LLM 倾向于用 Markdown 表格呈现对比数据，可读性强

### 6.2 弱项

1. **来源标注**（26%）—— 只有 1/4 的回复明确标注了数据来源 URL。虽然 search_web 找到了真实数据，但 LLM 在整合回答时常常"忘记"保留来源信息
2. **投档线/专业线区分**（51%）—— 系统提示词中强调了这个规则，但有一半的回复未在文本中明确区分二者。部分情况下因为题目本身不涉及（如纯政策咨询）
3. **免责提醒**（61%）—— 39% 的回复缺少"以官方为准"的提醒

### 6.3 改进建议

| 优先级 | 建议 | 预期效果 |
|:---:|------|------|
| 高 | 系统提示词加"每条录取数据必须标注来源 URL" | 来源标注率 26%→80%+ |
| 高 | 系统提示词加"涉及分数时必须注明是投档线还是专业线" | 区分率 51%→80%+ |
| 中 | 提示词加"回答末尾统一加免责提醒" | 免责率 61%→95%+ |
| 低 | 特殊类型/政策规则类增加知识库深度 | C 级率 5.5%→<2% |
