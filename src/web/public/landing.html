该项目是一个面向高考考生和家长的志愿填报咨询 Agent。它不讲废话，不编数据，它先找数据再给结论最后讲现实，像一个愿意讲真话的老师。点击开始体验吧！最后，感谢张雪峰团队，小石谈AI技对该项目的数据支撑。

项目总览
Wiki 知识库 1316 个 Markdown 文件，7 个目录：政策规则、省份数据、院校库（993 所）、专业库（176 个）、风格库、案例库、录取数据说明。Obsidian 格式，支持 [[双向链接]]，"00_全国高考地图"为总入口，四层深度阅读路径。

链接处理 — 裸引用（[[计算机]]）保留原样让 LLM 自行搜索；路径引用（[[01_政策规则/山东_2026]]）自动解析为可跳转文件路径。search_wiki 工具启动时构建 1274 个文件名索引，支持按路径精确读取，带路径遍历防护。
RAG 设计 原始素材来自 wiki 目录 Markdown，经 sync 脚本同步 staging，按标题切分，超 1200 字符按双换行再切，40 字符以下丢弃。DashScope embedding（1024 维）写入 Qdrant。

五集合隔离 — schools、majors、policies_rules、province_data、style_cases，共 10309 个向量。LLM 必须指定集合搜索，按 sourceFile 去重并回填完整原始文档。配套 search_wiki 工具按名索引、路径精确读取。
录取数据库 SQLite（gaokao_2025.db），三张表：school_scores（2.7 万条）、major_scores（87 万条）、province_lines（94 条），覆盖 990 所院校和全国 30+ 省级行政区。

数据维度 — 最低分、平均分、最高分、最低位次、招生计划数、控制线。索引建在 school、province、(school,province) 等高频路径，专业表额外加 major 模糊匹配索引。
工作流编排 ReAct 循环：用户提问后 LLM 流式思考并决定是否调用工具。tool_calls 逐个执行，结果注入对话历史，进入下一轮。最多 10 轮，超限后注入强制消息"不要再调用工具，直接回答"，最后做一次不带工具定义的调用。

四工具协同 — search_knowledge 语义模糊搜索、search_wiki 精确文件读取、search_data 结构化数据查询、search_web 实时网络搜索。系统提示词指导 LLM 同一轮内并行调用多个工具覆盖不同维度。
Agent 运行时 GaokaoAgent 串联消息构建器、工具执行器、ChatLoop 三个组件。消息构建每次叠三层（系统提示词→SQLite 历史含 tool_calls 反序列化→当前提问），全历史平铺；工具执行器四个工具统一实现 Tool 接口，signal 穿透底层；ChatLoop 每轮 SSE 流式调 LLM，tool_calls 跨 chunk 累积拼接，最多 10 轮超限强制汇总。

边界保护 — AbortSignal 贯全线，取消后从消息数组重建不丢数据；工具失败包装为错误字符串不中断；结果超一万字截断；存库前过滤特殊标签；消息保存用显式事务。
